## 1. 데모 데이터 상세·초기화

```bash
python seed.py            # 데모 상품 15개 + 계정 4개 + 사진 등록
python seed.py --reset    # 기존 데모 데이터 삭제 후 재등록
```

- 상품 사진은 카테고리에 맞는 무료 이미지(loremflickr 등)를 내려받으며,
  네트워크가 없으면 자동으로 플레이스홀더 이미지를 생성합니다.
- 데모 계정(비밀번호 공통 `Passw0rd1`): 판매자 `daangn_kim` / `market_lee` / `hanguk_park`,
  구매자 `buyer_choi`(잔액 넉넉).

## 2. 실제 결제(충전) — 토스페이먼츠

기본값은 `mock`(가상 PG)이라 별도 설정 없이 충전 흐름이 동작합니다.
실제 결제창으로 시연하려면 [토스페이먼츠](https://developers.tosspayments.com) 연동을 켭니다.

1. 토스페이먼츠 개발자센터에서 **테스트 API 키**(클라이언트 키 `test_ck_...`, 시크릿 키 `test_sk_...`)를 발급합니다.
2. 아래처럼 키를 환경변수로 넣어 실행합니다. (결제창은 HTTPS가 필요하므로 ngrok과 함께 쓰는 것을 권장 — 3번 참고)

```bash
PAYMENT_PROVIDER=toss \
TOSS_CLIENT_KEY=test_ck_xxxxxxxx \
TOSS_SECRET_KEY=test_sk_xxxxxxxx \
SESSION_COOKIE_SECURE=1 TRUST_PROXY=1 python app.py
```

3. 지갑 → **충전하기** → 금액 입력 → 토스 결제창에서 **테스트 카드**로 결제하면 잔액이 충전됩니다.

- **라이브 키**(`live_ck_...`)로 바꾸면 실제 결제가 됩니다. 운영에는 사업자 등록·심사가 필요하며,
  전환은 본인 책임 하에 진행하세요.
- 결제 보안: 승인 금액은 서버에 저장된 주문 금액과 일치할 때만 처리(위·변조 차단), 승인은
  `pending→paid` 조건부 UPDATE로 멱등 처리(이중 충전 방지), 토스 승인은 시크릿 키로
  서버-서버 확인(시크릿은 클라이언트에 노출되지 않음).

## 3. 외부에서 접속 A — ngrok

공인 IP·포트포워딩 없이 HTTPS 주소로 외부에 공개합니다(과제 슬라이드 권장 방식).
`ngrok http`는 로컬 `127.0.0.1:5000`으로 전달하므로 서버는 기본 주소 그대로 실행합니다.

```bash
# 1) 설치 (미설치 시)
sudo snap install ngrok        # 또는 https://ngrok.com/downloads/linux

# 2) 무료 계정 발급 후 authtoken 등록 (최초 1회)
#    https://dashboard.ngrok.com/get-started/your-authtoken
ngrok config add-authtoken <YOUR_AUTHTOKEN>

# 3) 앱 실행 (HTTPS 프록시에 맞춰 Secure 쿠키 + 프록시 신뢰)
SESSION_COOKIE_SECURE=1 TRUST_PROXY=1 python app.py

# 4) 새 터미널에서 터널 오픈
ngrok http 5000
```

출력되는 `https://xxxx.ngrok-free.app` 주소로 외부에서 접속합니다.
브라우저↔ngrok 구간이 HTTPS라 세션 쿠키·CSRF·소켓(WSS)이 안전하게 동작합니다.

## 4. 외부에서 접속 B — 같은 네트워크(LAN)

같은 공유기에 연결된 기기에서 접속할 때 사용합니다.

```bash
HOST=0.0.0.0 python app.py
# 접속 주소: http://<이 PC의 IP>:5000   (예: http://192.168.0.10:5000)
#  - IP 확인: hostname -I
#  - WSL에서 실행 시 Windows 방화벽/포트 프록시(netsh) 설정이 필요할 수 있음
```

> ⚠️ 평문(HTTP)이라 신뢰할 수 있는 사설망에서만 사용하세요. 인터넷 공개는 3번(ngrok)이나 5번(배포)을 쓰고,
> 공개 시 `ADMIN_PASSWORD`를 반드시 강력하게 지정하세요.

## 5. 실제 도메인으로 배포

`localhost`가 아닌 **고정 도메인 + HTTPS**로 서비스합니다. WebSocket(채팅) 때문에 프로덕션에서는
개발 서버 대신 **gunicorn + gevent-websocket 워커**로 실행합니다.
포함 파일: `wsgi.py`, `Dockerfile`, `Procfile`, `render.yaml`, `.dockerignore`.

> 참고: 최신 gunicorn(23+)에서 `-k eventlet` 워커가 **제거**되었습니다.
> (`Entry point ('gunicorn.workers', 'eventlet') not found` 오류)
> 따라서 gevent + gevent-websocket 조합을 사용합니다.

### 5-1. 로컬에서 프로덕션 모드로 실행

```bash
pip install -r requirements.txt gunicorn gevent gevent-websocket
SESSION_COOKIE_SECURE=1 TRUST_PROXY=1 PORT=8000 \
  gunicorn -k geventwebsocket.gunicorn.workers.GeventWebSocketWorker \
  -w 1 wsgi:app --bind 0.0.0.0:8000
# 확인: curl -I http://127.0.0.1:8000/   → 200 + Strict-Transport-Security 헤더
```

### 5-2. Docker

```bash
docker build -t mymarket .
docker run -p 8000:8000 -e ADMIN_PASSWORD='강력한비밀번호1' mymarket
```

### 5-3. PaaS로 무료 실도메인 받기 (예: Render.com)

1. 저장소를 GitHub에 push (이미 되어 있음).
2. Render 대시보드 → New → Blueprint → 저장소 선택 → `render.yaml` 자동 인식.
3. 배포되면 `https://<앱이름>.onrender.com` 같은 **실제 HTTPS 도메인**이 발급됩니다.
4. (선택) 커스텀 도메인은 Settings → Custom Domain에서 연결.

Railway·Fly.io·PythonAnywhere 등도 `Procfile`/`Dockerfile`을 인식해 동일하게 배포됩니다.

> ⚠️ 무료 티어는 파일시스템이 **비영속**이라 SQLite `market.db`와 업로드 사진이 재배포 시
> 초기화될 수 있습니다. 지속 운영에는 PostgreSQL + 오브젝트 스토리지 이관을 권장하며,
> `SECRET_KEY`는 반드시 환경변수로 고정하세요(미지정 시 인스턴스마다 랜덤 → 세션 무효화).

## 6. 운영 팁

```bash
# 실행 중인 서버 종료
#  - 서버를 띄운 터미널에서는 Ctrl+C 가 가장 안전
#  - 다른 터미널에서 종료할 때 (대괄호 트릭으로 자기 자신 매칭 방지)
pkill -f "[p]ython app.py"

# DB·업로드 사진 완전 초기화 (관리자 비번을 초기값으로 되돌릴 때도 사용)
rm -f market.db && rm -f static/uploads/*
python app.py            # 재실행 시 admin/Admin123! 로 재생성(최초 로그인 변경 강제)

# 관리자 비밀번호를 지정해서 새로 만들기
rm -f market.db && ADMIN_PASSWORD='새비밀번호1' python app.py

# 백그라운드 실행 로그 확인
tail -f app.log
```

## 7. 전체 환경변수

| 변수 | 기본값 | 설명 |
|---|---|---|
| `SECRET_KEY` | 랜덤 생성 | Flask 세션 서명 키. 미지정 시 랜덤 생성 후 `instance/secret_key`에 보관 |
| `ADMIN_PASSWORD` | `Admin123!` | 최초 관리자 계정 비밀번호 (미지정 시 최초 로그인에서 변경 강제) |
| `HOST` | `127.0.0.1` | 서버 바인딩 주소. LAN/외부 직접 노출 시 `0.0.0.0` |
| `PORT` | `5000` | 서버 포트 |
| `SESSION_COOKIE_SECURE` | `0` | `1`이면 Secure 쿠키 + HSTS 적용 (HTTPS/ngrok 환경 권장) |
| `TRUST_PROXY` | `0` | `1`이면 리버스 프록시의 `X-Forwarded-Proto/Host` 신뢰 (ngrok 등 앞단에 있을 때만) |
| `CORS_ORIGINS` | (없음) | 소켓 허용 오리진. 미지정 시 same-origin만 허용 |
| `IP_LOGIN_LIMIT` | `10` | IP당 5분 내 로그인 시도 허용 횟수 (자동화 테스트 시 상향) |
| `PAYMENT_PROVIDER` | `mock` | 충전 결제 수단. `mock`(가상 PG) 또는 `toss`(토스페이먼츠) |
| `TOSS_CLIENT_KEY` | (없음) | 토스 클라이언트 키(공개). `PAYMENT_PROVIDER=toss`일 때 필요 |
| `TOSS_SECRET_KEY` | (없음) | 토스 시크릿 키(서버 전용, 절대 노출 금지) |
