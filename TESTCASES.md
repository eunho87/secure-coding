# 테스트 케이스 명세 및 결과

Tiny Second-hand Shopping Platform — 실제 테스트(테스트 케이스 생성·입력 및 정상 동작 확인)

- 대상 커밋: `main` 최신
- 실행 환경: WSL2 Ubuntu, Python 3.12, 서버 `IP_LOGIN_LIMIT=1000 python app.py`
- 자동화: `tests/e2e_test.sh`(67) + `tests/extra_test.sh`(8) + `tests/finance_test.sh`(6) = **81개 전부 PASS**
- 아래 표의 "검증"은 자동화 스크립트가 실제 HTTP 요청/DB 조회로 확인한 항목이며,
  "수동"으로 표기된 항목은 브라우저 시연으로 확인한다.

실행 방법:
```bash
IP_LOGIN_LIMIT=1000 python app.py &
bash tests/e2e_test.sh
bash tests/extra_test.sh
bash tests/finance_test.sh
```

## 4. 데모 시나리오 (수동 시연 절차)

시드 데이터(`python seed.py`)가 등록된 상태에서 아래 순서로 "사고 파는" 전 과정을 시연할 수 있다.
데모 계정 비밀번호는 모두 `Passw0rd1`.

1. **(비로그인) 상품 구경** — 첫 화면 → "상품 구경하기" → 15개 상품을 로그인 없이 열람 (TC-P06)
2. **buyer_choi 로그인** — 잔액 3,000,000원 보유
3. **아이폰 13 상세** → 이미 등록된 500,000원 가격 제안 확인 (판매자 hanguk_park가 수락하면 그 가격으로 구매)
4. **다른 상품 안전거래 구매** — 예: "닌텐도 스위치" → 비밀번호 입력 → 결제 → 대금 에스크로 보관
5. **지갑 → 안전거래 현황** — 구매 확정 → 판매자에게 지급 → 거래 후기(별점) 작성
6. **hanguk_park 프로필** — 완료 거래 후기로 ★5.0 평점 표시 (버즈2 프로 판매 완료 이력)
7. **admin 로그인** — 관리자 콘솔 → 재무 감사에서 원장·잔액 무결성 확인, 신고/분쟁 관리
