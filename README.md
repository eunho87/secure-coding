# Tiny Second-hand Shopping Platform (Secure Coding)

시큐어 코딩을 적용한 중고거래 플랫폼입니다.
Flask + Flask-SocketIO + SQLite 기반으로, 실제 중고거래 서비스(당근마켓/번개장터 등)를 참고해
**에스크로 안전거래, 가격 제안(네고), 거래 후기, PG 결제 충전** 등 실거래 수준의 기능을 구현했습니다.

- GitHub: https://github.com/eunho87/secure-coding
- 개발 전 과정 보고서: **[REPORT.md](REPORT.md)** (요구사항→설계→구현→테스트→유지보수, 보안 약점 43건)
- 테스트 케이스 명세: **[TESTCASES.md](TESTCASES.md)**
- 외부 공개·실결제·배포 등 추가 실행 절차: **[실행방법.md](실행방법.md)**

## 주요 기능

| 분류 | 기능 |
|---|---|
| 유저 관리 | 회원가입, 로그인/로그아웃, **2단계 인증(TOTP)**, 사용자 프로필 조회(평점·후기), 마이페이지(소개글·비밀번호 변경) |
| 상품 | 상품 등록(카테고리·상품상태·**사진 최대 5장**), 수정/삭제(소유자 검증), 조회수, **찜하기/찜 목록**, 판매중→예약중→판매완료 상태 관리 |
| 검색 | 키워드(제목+설명), 카테고리/가격대 필터, 정렬(최신·저가·고가·인기순) |
| 거래 | 지갑(**PG 결제 충전**·이중기입 원장), 유저 간 송금(일일 한도), **에스크로 안전거래**: 결제 대금 플랫폼 보관 → 구매 확정 시 판매자 지급, 취소 요청/승인, 분쟁 시 관리자 중재 |
| 결제 | **PG 연동 충전**(가상 PG 기본 / 토스페이먼츠 테스트·라이브), 금액 서버 검증·멱등 크레딧·서버-서버 승인 |
| 금융 보안 | **이중 정산 방지**(멱등 정산), **1회용 폼 토큰**(이중 제출 방지), **거래 시 2FA 재인증**, **이상거래탐지(FDS)**, 잔액 원장 무결성 감사 |
| 네고 | **가격 제안** → 판매자 수락/거절 → 수락가로 자동 결제 |
| 후기 | 거래 완료 후 상호 별점(1~5)+코멘트, 프로필에 평균 평점 표시 |
| 소통 | 실시간 전체 채팅, 1:1 채팅(내역 저장), **알림함**(제안/거래/후기/송금 알림) |
| 신고/차단 | 사용자·상품 신고, 신고 3회 누적 상품 자동 차단, 5회 누적 유저 자동 휴면 |
| 관리자 | 사용자/상품/신고 관리, **에스크로 분쟁 중재**(지급/환불), **재무 무결성 감사**, 감사 로그 |

## 적용한 보안 요소 (요약)

- **비밀번호**: bcrypt 해시 + salt, 8자 이상·영문+숫자 정책
- **2단계 인증**: RFC 6238 TOTP (Google Authenticator 등 연동, 시도 횟수 제한)
- **인증/세션**: HttpOnly·SameSite 쿠키, 30분 만료, 세션 고정 방지,
  **비밀번호 변경 시 전체 기기 세션 무효화**(세션 버전), 로그인 5회 실패 시 계정 잠금 + **IP 단위 rate limit**
- **재인증**: 비밀번호 변경·송금·결제 시 비밀번호 재확인 + (2FA 사용자) TOTP 코드
- **거래 안전**: 에스크로 결제(직접 송금 방지), **원자적 잔액 처리**(단일 통로 `apply_balance`),
  **상태 조건부 멱등 정산**(이중 지급 방지), **이중기입 원장**, **1회용 폼 토큰**, **일일 한도 + FDS**, `balance CHECK(>=0)`
- **결제**: 승인 금액 서버 검증(위·변조 차단), 멱등 크레딧, 서버-서버 승인(시크릿 비노출)
- **CSRF**: 모든 POST 폼에 Flask-WTF CSRF 토큰
- **SQL Injection**: 전 쿼리 파라미터 바인딩, LIKE 이스케이프
- **XSS**: Jinja2 autoescape + 채팅 `textContent` 출력, CSP(nonce 기반)
- **입력 검증**: 사용자명/가격/정렬키/카테고리 등 화이트리스트 서버측 검증
- **채팅**: 소켓 세션 인증, 서버측 사용자명(사칭 방지), rate limit
- **파일 업로드**: 매직바이트 검증, 랜덤 파일명, 장수·용량 제한
- **관리자**: 기본 비밀번호 사용 시 최초 로그인에서 변경 강제
- **기타**: 보안 헤더(CSP/X-Frame-Options/HSTS 등), 커스텀 에러 페이지, 감사 로그, SECRET_KEY 랜덤 생성

발견·수정한 보안 약점 43건의 상세 내역은 [REPORT.md](REPORT.md) 3.3장 참고.

## 환경 설정

Ubuntu(WSL 포함) + Python 3.9 이상 기준입니다. 아래 둘 중 하나를 선택하세요.

### 방법 1: conda (miniconda/anaconda)

```bash
git clone https://github.com/eunho87/secure-coding
cd secure-coding
conda env create -f enviroments.yaml
conda activate secure_coding
```

### 방법 2: venv + pip

```bash
git clone https://github.com/eunho87/secure-coding
cd secure-coding
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 실행 방법

```bash
python app.py
```

- 서버 주소: **http://127.0.0.1:5000**
- 최초 실행 시 `market.db`(SQLite)가 자동 생성됩니다.
- 상품 목록·상세·검색은 **로그인 없이 누구나** 열람 가능(요구사항), 구매·채팅·찜·판매 등은 로그인 필요.

## 데모 데이터 등록 (채점·시연 권장)

빈 화면 대신 상품·계정·사진·잔액을 미리 채워, 바로 "사고 파는" 과정을 볼 수 있습니다.
서버가 DB를 만든 뒤(한 번 실행 후) 다른 터미널에서 실행하세요.

```bash
python seed.py          # 데모 상품 15개 + 계정 4개 + 사진 등록
```

- **데모 계정 (비밀번호 공통 `Passw0rd1`)**
  - 판매자: `daangn_kim`, `market_lee`, `hanguk_park`
  - 구매자: `buyer_choi` (잔액 넉넉 — 바로 구매 시연 가능)
- 시연 순서(구매·채팅·에스크로 등)는 [TESTCASES.md](TESTCASES.md) 4장에 정리되어 있습니다.

## 관리자 계정

최초 실행 시 자동 생성됩니다.

- 아이디: `admin`
- 비밀번호: 환경변수 `ADMIN_PASSWORD` 값 (미지정 시 `Admin123!`)
- **기본 비밀번호로 생성되면 최초 로그인 시 비밀번호 변경이 강제됩니다.**

```bash
# 권장: 실행 전 관리자 비밀번호 지정
ADMIN_PASSWORD='나만의강력한비밀번호1' python app.py
```

## 테스트 실행

기능·보안·금융을 실제 HTTP 요청/DB 조회로 검증하는 자동화 스크립트입니다.
로그인 IP rate limit이 자동화와 충돌하지 않도록 서버를 `IP_LOGIN_LIMIT=1000`으로 띄웁니다.

```bash
IP_LOGIN_LIMIT=1000 python app.py &     # 테스트용 서버 실행
bash tests/e2e_test.sh      # 기능 + 보안 E2E (67개)
bash tests/extra_test.sh    # 소켓 인증·업로드 검증·2FA (8개)
bash tests/finance_test.sh  # 금융: 이중정산·결제멱등·원장무결성 (6개)
```

**총 81개 테스트 전부 PASS.** 각 케이스 명세는 [TESTCASES.md](TESTCASES.md) 참고.
